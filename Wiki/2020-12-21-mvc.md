# MVC

## MVC

- 디자인 패턴 중 하나
- Model, View, Controller

## Model

- 애플리케이션의 정보, 데이터를 나타냄
- 데이터, 정보들이 가공을 책임직는 컴포넌트
- 규칙
  - 사용자가 편집하기 원하는 모든 데이터를 가지고 있어야한다.

## View

- 사용자 인터페이스 요소를 나타내준다.
- 데이터 및 객체의 입력, 출력을 담당
- 사용자들이 볼 수 있는 화면
- 규칙
  - 모델이 가지고 있는 정보를 따로 저장해서는 안된다.

## Controller

- 데이터와 사용자의 인터페이스 요소들을 잇는 다리역할
- 이벤트들을 처리하는 부분
- 규칙
  - 모델이나 뷰에 대해서 알고 있어야한다.
